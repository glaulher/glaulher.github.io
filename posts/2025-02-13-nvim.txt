<p>Tenho utilizado o Neovim como meu editor de código há um tempo.
No início, optei por configurá-lo manualmente, do zero.
No entanto, conforme fui adicionando mais plugins, comecei a enfrentar
dificuldades — alguns tiveram suas APIs modificadas,
enquanto outros foram descontinuados. Isso tornou a atualização com o Packer,
gerenciador de plugins que eu utilizava, uma tarefa problemática,
pois frequentemente resultava em falhas no sistema.</p>
<p>Em busca de uma solução mais estável, experimentei o LunarVim,
mas achei a configuração um pouco "bloated". Recentemente,
optei pelo Kickstart.nvim, uma configuração mínima e eficiente mantida por TJ Devries,
um dos desenvolvedores do Neovim e responsável por plugins essenciais
como Telescope e Plenary, que já vêm integrados ao Kickstart.
Até agora, a experiência tem sido excelente.</p>
<h3>Como instalar o Kickstart.nvim</h3>
<p>Se você deseja testar o Kickstart.nvim, siga os passos abaixo:</p>
<p>1 ➜ <strong>Instale o Neovim</strong></p>
<p>Se ainda não tem o Neovim instalado, baixe a versão mais recente no <a href="https://neovim.io/">site oficial</a> ou instale via gerenciador de pacotes:</p>
<pre><code class="language-sh"># No Debian/Ubuntu
sudo apt install neovim

# No Arch Linux
sudo pacman -S neovim

# No macOS (via Homebrew)
brew install neovim

# No openSUSE
sudo zypper install neovim
</code></pre>
<p>2 ➜ <strong><a href="https://github.com/nvim-lua/kickstart.nvim">Clone o Kickstart.nvim</a></strong></p>
<p>Para começar com a configuração mínima do Kickstart.nvim, basta clonar o repositório diretamente na pasta de configuração do Neovim:</p>
<pre><code class="language-sh">git clone --depth=1 https://github.com/nvim-lua/kickstart.nvim.git ~/.config/nvim
</code></pre>
<p>3 ➜ <strong>Abra o Neovim e finalize a instalação</strong></p>
<p>Execute o Neovim e os plugins serão instalados automaticamente:</p>
<pre><code class="language-sh">nvim
</code></pre>
<p>Aguarde a instalação ser concluída e reinicie o editor para garantir que tudo esteja funcionando corretamente.</p>
<p>4 ➜ <strong>Personalize sua configuração</strong></p>
<p>O Kickstart.nvim é apenas uma base mínima. Você pode personalizá-lo editando o arquivo <code>init.lua</code> localizado em <code>~/.config/nvim/init.lua</code>. Se precisar adicionar novos plugins, edite a parte onde o <code>lazy.nvim</code> está configurado.</p>
<p>5 ➜ <strong>Checkhealth</strong><br>
Rode o comando :<code>checkhealth</code> no nvim e verifique a saída, instalando as ferramentas que faltam:</p>
<pre><code class="language-lua">
